Setup — dostęp przez domenę (Let's Encrypt)

{{if not .HasDomain}}
Brak skonfigurowanej domeny. Uruchom lbpanel z flagą --domain lbpanel.twojadomena.pl lub ustaw zmienną środowiskową LBPANEL_DOMAIN=lbpanel.twojadomena.pl.
{{end}}

Jak to działa

lbpanel nasłuchuje na porcie 4040 HTTPS z certyfikatem self-signed.
Dostęp przez IP (https://192.168.1.1:4040) działa od razu — przeglądarka wyświetli ostrzeżenie o certyfikacie, które należy zaakceptować raz.

Aby uzyskać pełny HTTPS z Let's Encrypt przez domenę, Caddy musi działać jako reverse proxy przed panelem na portach 80/443. Poniżej gotowy Caddyfile do wklejenia.

Tryb 1 — dostęp przez IP (self-signed, gotowe od razu)

https://<IP_SERWERA>:4040

Przeglądarka pokaże "Twoje połączenie nie jest prywatne" — kliknij "Zaawansowane → Przejdź mimo to". Certyfikat jest self-signed ale połączenie jest szyfrowane.

{{if .HasDomain}}

Tryb 2 — domena {{.Domain}} z Let's Encrypt

Wklej do /etc/caddy/Caddyfile na serwerze lb01, następnie: systemctl reload caddy 

# /etc/caddy/Caddyfile — fragment dla lbpanel
# Wymaga: DNS A/AAAA dla {{.Domain}} wskazuje na ten serwer

# Przekierowanie www → bez www
www.{{.Domain}} {
    redir https://{{.Domain}}{uri} permanent
}

# Panel z Let's Encrypt (Caddy pobierze certyfikat automatycznie)
{{.Domain}} {
    # Reverse proxy do lbpanel (self-signed backend — skip verify)
    reverse_proxy https://127.0.0.1:4040 {
        transport http {
            tls_insecure_skip_verify
        }
    }

    # Logowanie dostępu do panelu
    log {
        output file /var/log/caddy/lbpanel-access.log {
            roll_size 10mb
            roll_keep 5
        }
    }

    # Opcjonalne: ogranicz dostęp tylko z wybranych IP
    # @blocked not remote_ip 1.2.3.4/32 192.168.0.0/24
    # respond @blocked "Forbidden" 403
}

DNS — co ustawić w Cloudflare / rejestratorze

TypNazwaWartośćProxy
A {{.Domain}} <IP_SERWERA_LB01> ⚠ OFF (szara chmurka — Let's Encrypt wymaga bezpośredniego dostępu)
CNAME www.{{.Domain}} {{.Domain}} ⚠ OFF

Po uzyskaniu certyfikatu możesz włączyć Cloudflare proxy (pomarańczowa chmurka) i zmienić SSL/TLS mode na Full (strict). Wtedy Caddy musi mieć certyfikat CF Origin lub Let's Encrypt — oba działają.

Sprawdzenie po konfiguracji

# Test HTTP challenge (musi odpowiedzieć 200/301 przed włączeniem CF proxy)
curl -I http://{{.Domain}}/.well-known/acme-challenge/test

# Status certyfikatu Caddy
caddy trust
journalctl -u caddy -n 30 --no-pager

# Test panelu przez domenę
curl -k https://{{.Domain}}/health

Konfiguracja domeny

Ustaw domenę w /opt/lbpanel/lbpanel.env:

Następnie: systemctl restart lbpanel
